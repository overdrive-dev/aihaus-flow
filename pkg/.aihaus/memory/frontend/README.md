# Frontend Memory

This directory holds UI and client-side knowledge accumulated from this
repository.

Fresh installs intentionally start with no inherited frontend facts.

## What Goes Here

- Component naming, file layout, and prop conventions.
- State-management and side-effect boundaries.
- Styling, token, and responsive-layout conventions.
- Frontend-specific build, SSR, hydration, and browser gotchas.
- UI test conventions and accessibility assertions.

## What Does Not Go Here

- Cross-cutting project rules.
- Backend-specific patterns.
- Reviewer output.
- Unanswered business questions.
