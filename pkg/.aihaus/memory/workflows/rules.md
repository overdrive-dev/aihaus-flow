# Workflow Rules

Use this file for repository-specific workflow rules that agents must follow.

Examples:

- source-of-truth policy for the external tracker, board, or local task files,
- required gates before `review-dev`, `human-review`, or production movement,
- branch, PR, deploy, and smoke-test requirements,
- rules for blocking one task without blocking the whole goal,
- rules for writing one planning blocker per task instead of batch questions,
- rules for when sync debt is acceptable.

Rules in this file should be actionable. If a rule is no longer current, append
a dated correction instead of silently deleting history.
