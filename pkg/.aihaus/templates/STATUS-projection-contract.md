# STATUS.md — Projection Contract

**Status:** Canonical — ADR-004 projection invariant (amendment to ADR-001).
**Introduced:** M003 (2026-04-14).

## Role

`STATUS.md` is the **derived projection** of `RUN-MANIFEST.md`'s `Metadata.phase` field. It answers "what phase is this milestone / draft / feature in?" with one word. Readers (`aih-milestone` gathering + `annexes/execution.md`, `aih-resume`, humans scanning the filesystem) can check STATUS.md in O(1) without parsing the full manifest.

**`STATUS.md` is NEVER the authoritative state.** `RUN-MANIFEST.md` is truth. If the two disagree, RUN-MANIFEST wins and STATUS.md is regenerated by `phase-advance.sh`.

## Contract

Every STATUS.md file SHOULD have three lines:

```text
<phase>
<!-- DERIVED FROM RUN-MANIFEST.md — DO-NOT-EDIT-MANUALLY (phase-advance.sh is the sole writer per ADR-004) -->
<!-- last_updated: <iso-8601-utc> -->
```

- **Line 1:** the phase, one of `gathering | planning | ready | running | complete | paused`.
- **Line 2:** the DO-NOT-EDIT marker. Presence tells future readers + tooling that this file is projection-owned.
- **Line 3:** ISO-8601 UTC timestamp of the last `phase-advance.sh` invocation.

## Writers

The **ONLY** writer of STATUS.md post-M003 is `phase-advance.sh`. Skills must NOT inline-write STATUS.md:

```bash
# WRONG (pre-M003 pattern — rewired in M003 story 13; historical reference)
echo "ready" > STATUS.md

# RIGHT (ADR-004 pattern)
bash .aihaus/hooks/phase-advance.sh --to ready --dir .aihaus/milestones/drafts/<slug>/
```

`phase-advance.sh` additionally:
- Refuses if `Invoke stack` non-empty in sibling `RUN-MANIFEST.md` (active invocation blocks transition).
- Backs up handwritten (non-DERIVED-marked) STATUS.md to `.handwritten.bak` before overwriting.
- Updates `PLAN.md` / `CONTEXT.md` Status frontmatter idempotently.
- Uses atomic `tmp + mv` (with Python `os.replace` fallback on OneDrive paths).

## Readers

Any code or agent may **read** STATUS.md freely. Typical readers:
- `aih-milestone/SKILL.md` + `annexes/execution.md` — Glob for `.aihaus/milestones/drafts/*/STATUS.md` to discover gathering/ready drafts and dispatch execution.
- `project-analyst` agent — `--refresh-active-milestones` reads STATUS.md per draft.
- `aih-resume/SKILL.md` — reads during recovery.

When reading, tolerate legacy single-line form (no DO-NOT-EDIT marker, no last_updated). Those are pre-M003 or hand-written drafts; the phase is still the first line.

## Migration

Existing pre-M003 STATUS.md files (single-line `gathering` / `ready`) remain valid. The next `phase-advance.sh` invocation upgrades them in place — no explicit migration needed.

The first invocation against a legacy STATUS.md backs it up to `STATUS.md.handwritten.bak` before overwriting (safe rollback).

## Non-goals

- STATUS.md carries no phase-history or timestamps beyond `last_updated`. The history lives in RUN-MANIFEST.md's Progress Log and Story Records.
- STATUS.md does not track the Invoke stack. That lives in RUN-MANIFEST.md's `## Invoke stack` section.
- STATUS.md is NOT human-editable. Humans who want to force a phase change edit `phase-advance.sh` or run it directly.
