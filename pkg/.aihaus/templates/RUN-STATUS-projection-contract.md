# RUN-STATUS.md ‚Äî Projection Contract

**Status:** Canonical ‚Äî ADR-M019-A parallel-projection invariant (extension of ADR-004).
**Introduced:** M019 (2026-05-01).
**Governing ADR:** ADR-M019-A (`pkg/.aihaus/decisions.md`).

<!-- DO-NOT-EDIT-MANUALLY ‚Äî this template is installed verbatim by install.sh into target repos -->

## Role

`RUN-STATUS.md` is the **derived projection** of `RUN-MANIFEST.md`'s story-level progress fields. It answers "which stories are done / in-flight / pending?" at a glance, in a format stable for `tail -F` consumers (VS Code on a second monitor, sidecar terminals). Readers can poll RUN-STATUS.md in O(1) without parsing the full manifest.

**`RUN-STATUS.md` is NEVER the authoritative state.** `RUN-MANIFEST.md` is truth. If the two disagree, `RUN-MANIFEST.md` wins; `RUN-STATUS.md` is regenerated by the next `manifest-append.sh` mutation.

### Phase vs progress separation (D1 spirit-justification)

STATUS owns phase only; RUN-STATUS owns progress only ‚Äî non-overlapping projections of disjoint state. (`STATUS.md` and `RUN-STATUS.md` respectively.)

A reader asking "what phase is the milestone in?" reads `STATUS.md`. A reader asking "which stories are done / in-flight / pending?" reads `RUN-STATUS.md`. **Never both at once.** The two projection files are non-overlapping derivatives of the same source-of-truth (`RUN-MANIFEST.md`), governed by ADR-M019-A.

| Field | STATUS.md | RUN-STATUS.md |
|---|---|---|
| Phase (`gathering`/`planning`/`ready`/`running`/`complete`/`paused`) | **YES ‚Äî sole source** | informational mirror only (may briefly lag) |
| Story progress (which stories done/in-flight/pending) | NO | **YES ‚Äî sole source** |
| Slice grid `[‚úì][‚Üí][ ]` | NO | **YES ‚Äî sole source** |
| Recent activity log | NO | **YES ‚Äî sole source** |
| Last `phase-advance.sh` invocation timestamp | YES | NO |
| Last `manifest-append.sh` mutation timestamp | NO | YES |

## Contract

Every `RUN-STATUS.md` file MUST have:

```markdown
<!-- DERIVED FROM RUN-MANIFEST.md ‚Äî DO-NOT-EDIT-MANUALLY (manifest-append.sh is the sole writer per ADR-M019-A) -->
<!-- last_updated: <iso-8601-utc> -->

# RUN-STATUS ‚Äî <ms-id>

**Phase:** <phase>          ‚Üê informational ONLY; canonical phase lives in STATUS.md
**Stories:** <done> done / <in-flight> in-flight / <pending> pending (<total> total)
**Current:** S<id> ‚Üí
**Agents in flight:** <n>
**SHA:** <7-char>

## Slice grid

S01 [‚úì]  S02 [‚úì]  S03 [‚Üí]  S04 [ ]  S05 [ ]
...

## Slice tree

- S01 ‚úì  <slug>  done <iso8601>  sha:<7-char>
- S02 ‚úì  <slug>  done <iso8601>  sha:<7-char>
- S03 ‚Üí  <slug>  in-flight (since <iso8601>)
- S04    pending
...

## Recent activity

- <iso8601> ‚Äî <row from ## Story Records or ## Checkpoints>
...
```

- **Header comment:** the `DERIVED FROM` + `DO-NOT-EDIT-MANUALLY` marker. Presence tells future readers + tooling that this file is projection-owned.
- **`last_updated` comment:** ISO-8601 UTC timestamp of the last `manifest-append.sh` invocation that regenerated this file.
- **Phase line:** informational mirror of `Metadata.phase`. Canonical phase lives in `STATUS.md`; this field may briefly lag.
- **Slice grid:** `[‚úì][‚Üí][ ]` ASCII cells, 5 per row, one cell per story. At-a-glance progress without reading the tree.
- **Slice tree:** one line per story with state, slug fragment, completion timestamp, and SHA (done stories) or in-flight/pending label.
- **Recent activity:** last N rows from `## Story Records` (and `## Checkpoints` if present), mtime-sorted descending. N = 10 by default; implementer may raise to 20 for large milestones.

## Writers

The **ONLY** writer of `RUN-STATUS.md` post-M019 is `manifest-append.sh`. No other script, skill, or agent may write this file directly:

```bash
# WRONG ‚Äî direct write, bypasses single-writer discipline
echo "..." > RUN-STATUS.md

# RIGHT ‚Äî manifest-append.sh regenerates RUN-STATUS.md atomically inside its critical section
bash .aihaus/hooks/manifest-append.sh --field story-record --payload "..."
```

`manifest-append.sh` regenerates `RUN-STATUS.md`:
- After every mutating case: `story-record`, `invoke-push`, `invoke-pop`, `progress-log`, `phase`, `status`, `checkpoint-enter`, `checkpoint-exit`.
- INSIDE the existing `coarse + inner` lock, AFTER `RUN-MANIFEST.md` atomic-rename completes and BEFORE lock release.
- Using atomic `tmp + mv` (with Python `os.replace` fallback on OneDrive / Dropbox / iCloud paths verbatim from `phase-advance.sh:170-185`).
- Fail-safe: any regen error returns 0 and leaves `RUN-MANIFEST.md` intact (NFR-005). The next mutation re-attempts.

### Slice total resolution order

The denominator in `**Stories:** N done / M in-flight / P pending (T total)` is resolved in this order:

1. **Primary:** count of `stories/S*.md` files in the milestone directory (file-count is always present post-scaffold).
2. **Secondary:** `PRD.md` `## In Scope ‚Äî N stories` header, if PRD is present.
3. **Last resort:** `?` (emitted when neither source is available; treated as a temporary degraded state).

## Readers

Any code or agent may **read** `RUN-STATUS.md` freely. Typical readers:
- `tail -F RUN-STATUS.md` in a sidecar terminal ‚Äî primary use case; VS Code on second monitor.
- `statusline-milestone.sh` ‚Äî may optionally read RUN-STATUS.md for slice grid rendering (M020 candidate; not required post-M019).
- `project-analyst` agent ‚Äî reads for active milestone summary without parsing the full manifest.
- Humans inspecting "where am I and what's left?" at a glance.

When reading, tolerate absence (file may not exist on milestones that pre-date M019; generated on first post-M019 mutation).

## Migration

Pre-M019 milestones do not have `RUN-STATUS.md`. The first mutating `manifest-append.sh` call on an active milestone post-M019 organically generates it. No explicit migration needed. Pre-existing milestones without `RUN-STATUS.md` are acceptable.

## Non-goals

- **No phase field as sole source.** `RUN-STATUS.md` MAY mirror phase informally, but `STATUS.md` is the canonical phase source. A reader that needs authoritative phase MUST read `STATUS.md`, not `RUN-STATUS.md`.
- **No security-token redaction.** See threat-model paragraph below. Discipline, not enforcement.
- **No retroactive regeneration.** Pre-existing milestones without `RUN-STATUS.md` generate on first post-M019 mutation.
- **No 4-bucket disambiguator in the slice grid.** The `[‚úì][‚Üí][ ]` 3-state grid ships in M019. A 4th state (e.g., `[!]` for stalled) is deferred to M020 and is contingent on cadence-delta instrumentation data.
- **Multi-worker progress.** Multi-worker progress lives in the slice grid `[‚úì][‚Üí][ ]`, not the single arrow. The `S<id>‚Üí` arrow shows the latest in-file row of `## Story Records` only; under interleaved/worktree execution (M017+) the arrow is informational, not authoritative.
- **`MTIME-STALE?<N>m` false-positive class (R3).** Legitimate long implementer tool sequences (between file checkpoints) can cause the `MTIME-STALE?<N>m` stall label in `statusline-milestone.sh` to fire without an actual stall. This is an accepted false-positive class; M020 candidate work tunes the threshold from cadence-delta instrumentation (FR-005). `RUN-STATUS.md` readers must tolerate this label when present.

### Threat-model Non-goal (D7 ‚Äî canonical text)

> **No secrets in story slugs / commit messages.** RUN-STATUS.md exposes story
> slugs, commit SHAs, agent activity timestamps, and recent-activity log entries.
> The file is intentionally human-readable and lives at a path that may be
> auto-synced by OneDrive / Dropbox / iCloud. Treat its contents as externally
> readable. Sensitive content (API keys, customer-identifying strings, internal
> URLs) MUST NOT appear in story slug names, commit messages, or any field that
> flows into the `## Story Records` rows. This is **discipline, not enforcement**:
> M019 ships no redaction layer; the projection is a faithful mirror of the
> manifest. **Multi-worker progress lives in the slice grid `[‚úì][‚Üí][ ]`, not
> the single arrow** ‚Äî the `S<id>‚Üí` arrow shows the latest in-file row of `##
> Story Records` only; under interleaved/worktree execution (M017+) the arrow
> is informational, not authoritative. **`MTIME-STALE?<N>m` false-positive
> class:** legitimate long implementer tool sequences (between file
> checkpoints) can trip the label without an actual stall. M020 candidate work
> tunes the threshold from cadence-delta instrumentation (FR-005).
