# State File — `.aihaus/.calibration`

Companion doc to `/aih-calibrate` SKILL.md Phase 4 and `/aih-update`'s
`restore_calibration()`. Describes the sidecar that lets calibration
survive package refreshes.

M009 shipped **schema v1** (effort-only per-agent overrides). M010 ships
**schema v2** additively — new cohort-level fields + per-agent `model:`
overrides. v1 sidecars keep restoring byte-identically via the legacy
dispatch; migration to v2 is write-once at the next `/aih-calibrate`
invocation.

Authoritative decisions: ADR-M009-A (sidecar ownership, preserved) +
ADR-M010-A (cohort taxonomy, schema v2) in `.aihaus/decisions.md`.

## Purpose, location, ownership (ADR-M009-A — preserved verbatim)

Sidecar survives `/aih-update` by living outside the refreshed
subdirectories (refresh loop only touches `skills/`, `agents/`, `hooks/`,
`templates/`). Records absolute restore targets so `/aih-update` can
re-apply `effort:` + (since M010) `model:` frontmatter edits and
`permissions.defaultMode`.

- **Location:** `.aihaus/.calibration` — flat file at `.aihaus/` root
  alongside `.install-mode`, `.install-platform`, `.install-source`,
  `.version`.
- **User-owned, never committed.** Written by `/aih-calibrate` Phase 4
  after commit + gate pass (never on `--dry-run` / self-revert fired).
  `git add .aihaus/.calibration` is forbidden; dogfood `.gitignore:19`
  covers it.
- **Derived state — safe to delete.** Missing sidecar = restore silent
  no-op. Regenerated by next `/aih-calibrate` invocation.
- **Not source of truth.** Git history (`chore(calibrate):` commits) is
  authoritative; sidecar is a re-application target.

## Schema v1 (legacy — read-only on v0.14.0+)

```
# aihaus calibration state — managed by /aih-calibrate, consumed by /aih-update
# Schema: v1
# This file is USER-OWNED and derived state. Safe to delete (regenerate via
# /aih-calibrate --inspect). Do not commit.

schema=1
permission_mode=bypassPermissions
last_preset=cost-optimized
last_commit=920cf48

# Per-agent effort overrides. Format: <agent-basename>=<effort-level>
analyst=high
architect=xhigh
implementer=high
```

### v1 fields

- **`schema=1`** — mandatory first non-comment line; dispatches to the
  legacy effort-only restore loop.
- **`permission_mode`** — absolute restore target for
  `permissions.defaultMode` in `.claude/settings.local.json`.
- **`last_preset`** — `cost-optimized | balanced | quality-first |
  auto-mode-safe | custom`.
- **`last_commit`** — short SHA of the `chore(calibrate):` commit.
- **`<agent-basename>=<effort-level>`** — one per calibrated agent.
  Effort level one of `low | medium | high | xhigh | max`.

v0.14.0+ readers keep v1 sidecars working forever — legacy dispatch is
byte-identical to v0.13.0 behavior. Only new writes use `schema=2`.

## Schema v2 (current — written by v0.14.0+)

```
# aihaus calibration state — managed by /aih-calibrate, consumed by /aih-update
# Schema: v2 — cohort taxonomy + per-agent (model, effort) overrides
# This file is USER-OWNED and derived state. Safe to delete. Do not commit.

schema=2
permission_mode=bypassPermissions
last_preset=cost-optimized
last_commit=deadbee

# Cohort-level — written per cohort on preset/--cohort apply.
# value=custom → non-uniform; per-agent lines take over.
cohort.planner.model=opus
cohort.planner.effort=high
cohort.doer.model=sonnet
cohort.doer.effort=high
cohort.verifier.model=sonnet
cohort.verifier.effort=medium
# :adversarial intentionally absent — preset-immune (filter at write time).

# Per-agent overrides — applied AFTER cohort-level (overrides win).
# v1-compat effort-only grammar:
architect=xhigh
planner=xhigh
product-manager=xhigh
roadmapper=xhigh
# v2 per-agent model grammar (dotted key):
architect.model=opus
planner.model=opus
```

### v2 fields

- **`schema=2`** — mandatory first non-comment line; dispatches to v2
  cohort-aware restore loop.
- **`permission_mode`** — unchanged from v1.
- **`last_preset`** — `cost-optimized | balanced | quality-first |
  auto-mode-safe | custom`. Unchanged from v1.
- **`last_commit`** — short SHA. Unchanged from v1.
- **NEW** `cohort.<name>.model=<opus|sonnet|haiku|custom>` — one per
  cohort. Accepted cohort names: `planner`, `doer`, `verifier`,
  `adversarial` (no colon on disk; `:` is CLI ergonomics only — keeps
  `IFS='='` parser simple). `custom` signals non-uniform — restore skips
  the cohort-level mutation for that field and defers to per-agent
  override lines.
- **NEW** `cohort.<name>.effort=<low|medium|high|xhigh|max|custom>` — one
  per cohort. Same `custom`-defer semantic.
- **Per-agent effort** `<agent-basename>=<effort-level>` — retained from
  v1; used for agents deviating from their cohort default (recorded on
  `--agent` escape-hatch or cohort migration `custom`-fallback).
- **NEW** per-agent model `<agent-basename>.model=<m>` — same role for
  model axis. `.model` suffix is the grammar disambiguator.

**Accepted cohort names (4):** `planner`, `doer`, `verifier`,
`adversarial`. Unknown cohort values warn + skip (forward-compat).

## Schema-gate behavior (hard boundary — ADR-M009-A preserved)

Restore reads `schema=` field first:

- `schema=1` → legacy v1 handler (effort-only per-agent; byte-identical
  to v0.13.0 behavior).
- `schema=2` → v2 handler (cohort-level apply first, per-agent overrides
  second — overrides win via apply order).
- any other value → `warn: unknown .calibration schema='X' — skipping
  restore` + return 0. No partial apply.

v2 readers MUST refuse to interpret a v1 sidecar as v2 and vice versa.
This is the schema-drift mitigation called out in analyst § Risks
(high-severity): mixed-version sidecars never exist because both readers
bail on unknown versions.

## Absolute-restore semantic (ADR-M009-A — preserved verbatim)

Recorded values are verbatim restore targets, not deltas. Parse invariants
inherited from v1: values MUST NOT contain `=`; CRLF-stripped before use;
blank + `#`-prefixed lines skipped; whitespace-only values skipped
defensively; unknown keys silently ignored for forward-compat. To track
new package defaults after an upgrade, re-run `/aih-calibrate --preset
<name>` — rewrites the sidecar with current distribution.

## v1 → v2 Migration (D-4 auto-infer; `custom` fallback)

First `/aih-calibrate` invocation on v0.14.0+ with a `schema=1` sidecar
present triggers migration. Write-once at the calibrate point — never
auto-rewritten by `/aih-update` alone (preserves ADR-M009-A user-owned
semantic).

```
for each cohort in [:planner, :doer, :verifier, :adversarial]:
  members = cohorts.md lookup
  if all members share same (model, effort) in current frontmatter:
    write cohort.<name>.model=<m> + cohort.<name>.effort=<e>
    # uniform → no warning
  elif cohort == :adversarial:
    # Expected non-uniform by design (2 baseline tiers — plan-checker /
    # contrarian at (opus, max); reviewer / code-reviewer at (opus, high)).
    write cohort.adversarial.model=custom + cohort.adversarial.effort=custom
    preserve each member as <agent>.model + <agent>=<effort>
    # NO warning — non-uniformity is ship default
  else:
    write cohort.<name>.model=custom + cohort.<name>.effort=custom
    preserve each deviating member as <agent>.model + <agent>=<effort>
    emit loud `!!` warning naming the cohort + remedy
```

**`!!` warning example (only when condition met, cohort ≠ `:adversarial`):**

```
!!  Cohort :planner is non-uniform — recorded as custom with per-agent preservation.
!!  Run /aih-calibrate --preset balanced to normalize.
```

The warning names the cohort(s) in custom state and includes the
`/aih-calibrate` command the user should run to normalize. Not silent —
ADR-M009-A absolute-restore semantic depends on the user understanding
what was preserved vs. what fell through.

## Adversarial preset-immunity (ADR-M010-A supersedes ADR-M008-C)

`:adversarial` cohort members (`plan-checker`, `contrarian`, `reviewer`,
`code-reviewer`) appear in v2 sidecars only on explicit user invocations:
`--cohort :adversarial --model X --effort Y` (both axes + literal-word
`adversarial` confirmation) or `--agent <member> --model X --effort Y`.
**Preset runs MUST NOT emit any `cohort.adversarial.*` line or per-agent
entry for an adversarial member** — filter lives at write time in
SKILL.md Phase-4 step 20. Preserves ADR-M008-C semantic under the
expanded 4-agent list; restore honors whatever is recorded.

## CRLF handling (Windows-compat — inherited from v1)

Bash: `value="${value%$'\r'}"` strips trailing `\r`. PowerShell:
`-replace "\r", ""` applied to every value; `Set-Content -Encoding UTF8
-NoNewline` on output frontmatter to match `sed -i.bak` byte-identically.

## Auto-mode-safe side-effect warning (preserved from v1)

When `last_preset=auto-mode-safe`, restore prints a loud `!!` block
pointing at `/aih-calibrate --preset auto-mode-safe` to replay side
effects the sidecar cannot auto-restore: removal of `permissionMode:
bypassPermissions` from worktree agents + widening of
`auto-approve-bash.sh` SAFE_PATTERNS. Re-run the preset to reapply.

## Migration path — pre-v0.13 hand edits (legacy, unchanged)

Pre-v0.13 users had no `/aih-calibrate`; first post-v0.13 update finds no
sidecar → early-return no-op. Run `/aih-calibrate --inspect` before the
first post-v0.13 update to snapshot state, then re-apply per-agent or
preset after.
