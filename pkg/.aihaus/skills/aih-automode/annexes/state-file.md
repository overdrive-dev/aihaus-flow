# State File — `.aihaus/.automode`

Companion doc to `/aih-automode` SKILL.md. Describes the sidecar that records
permission-mode state independently of the effort sidecar (`.aihaus/.effort`).

## Purpose, location, ownership

The `.automode` sidecar is the authoritative record of whether auto-mode has
been intentionally enabled in this repo. It is NOT a live reflection of
`settings.local.json` — if the two drift, `/aih-automode --status` reports
the sidecar value and warns about the drift. Run `/aih-automode --enable` or
`/aih-automode --disable` to reconcile.

- **Location:** `.aihaus/.automode` — flat file at `.aihaus/` root alongside
  `.install-mode`, `.install-platform`, `.install-source`, `.version`.
- **User-owned, never committed.** Written by `/aih-automode` only (never by
  `/aih-update` refresh loop directly). `git add .aihaus/.automode` is
  forbidden. The file is gitignored — see Gitignore section below.
- **Derived state — safe to delete.** Missing file treated as `enabled=false`.
  Regenerated by next `/aih-automode --enable` or `--disable` invocation.
- **Not source of truth for settings.local.json.** The sidecar records intent;
  `settings.local.json` reflects the applied state. If they drift (e.g. manual
  edit or partial restore), `/aih-automode --status` reports the sidecar value
  and emits a drift warning.

## Schema v1 (current — written by v0.16.0+)

```
enabled=true
last_enabled_at=2026-04-17T12:34:56Z
```

**Fields:**

| Field              | Values             | Required? | Description |
|--------------------|--------------------|-----------|-------------|
| `enabled`          | `true` \| `false`  | Yes       | Whether auto mode is currently enabled. Missing file = `enabled=false` (implicit default). |
| `last_enabled_at`  | ISO 8601 UTC       | Optional  | Timestamp of the last successful `--enable` invocation. Missing value prints `never` in `--status` output. |

No schema version number. v1 is implicit; a future v2 will add a `schema=2`
line while v1 readers treat its absence as v1.

**Format:** 2 lines, KEY=VALUE shell-syntax, UTF-8, LF line endings.

Example (enabled):
```
enabled=true
last_enabled_at=2026-04-17T12:34:56Z
```

Example (disabled):
```
enabled=false
last_enabled_at=2026-04-17T12:34:56Z
```

Note: `last_enabled_at` is preserved even after `--disable` so `--status`
can report the last time auto mode was active.

## Gitignore coverage

**This repo (aihaus-flow):** `.gitignore:19` contains `/.aihaus/` which
covers everything under `.aihaus/` including `.automode` and `.effort`.
Verify: `git check-ignore .aihaus/.automode` returns exit 0.

**User installs (target repos where users commit `.aihaus/`):** If a user's
repo does NOT wholesale-gitignore `.aihaus/` (they commit `project.md`,
`decisions.md`, `knowledge.md`), they must add explicit sidecar lines to
avoid accidentally committing the automode sidecar:

```bash
echo '.aihaus/.automode' >> .gitignore
echo '.aihaus/.effort' >> .gitignore
```

Similarly for the `.calibration.v2.bak` backup left by v2→v3 migration:
```bash
echo '.aihaus/.calibration.v2.bak' >> .gitignore
```

**Migration note for v0.15.0 users:** If your v0.15.0 gitignore had an exact
`.aihaus/.calibration` line (rather than the `/.aihaus/` wildcard), that
explicit line does NOT cover `.aihaus/.effort` post-M012 rename. Add the
`.aihaus/.effort` line above. The `.aihaus/.calibration.v2.bak` backup is
similarly not covered by the old explicit line.

## `--status` output contract

When `/aih-automode --status` runs:

- **Sidecar present:** prints both fields verbatim:
  ```
  enabled=true
  last_enabled_at=2026-04-17T12:34:56Z
  ```
- **Sidecar absent:** prints:
  ```
  enabled=false
  last_enabled_at=never
  ```
- **Drift detected:** if `settings.local.json` `permissions.defaultMode`
  disagrees with `enabled`, appends:
  ```
  !! drift: sidecar says enabled=<X> but settings.local.json defaultMode=<Y>
     Run /aih-automode --enable or --disable to reconcile.
  ```

## Idempotence contract

`/aih-automode --enable` is idempotent and must re-apply ALL side effects on
every call:

1. Check plan/provider/version eligibility (pre-check).
2. Display 4-caveat block (from `annexes/caveat-block.md`).
3. Prompt for literal word `auto`.
4. Merge `defaultMode=auto` into `settings.local.json` via `merge-settings.sh`.
5. Strip `permissionMode: bypassPermissions` from `implementer.md`,
   `frontend-dev.md`, `code-fixer.md` — even if already stripped (no-op safe).
6. Write `.automode` with `enabled=true` + `last_enabled_at=<iso>`.

Rationale: `/aih-update`'s refresh loop restores worktree agent templates
(including `permissionMode: bypassPermissions`). Re-running `--enable` after
`/aih-update` must re-strip the restored frontmatter — R5 mitigation,
FR-S04. There is no "first-enable-only" guard.
