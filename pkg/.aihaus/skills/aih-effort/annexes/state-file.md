# State File — `.aihaus/.effort`

Companion doc to `/aih-effort` SKILL.md Phase 4 and `/aih-update`'s
`restore_effort()`. Describes the sidecar that lets calibration survive
package refreshes.

M009 shipped **schema v1** (effort-only per-agent overrides, file `.calibration`).
M010 shipped **schema v2** additively — cohort-level fields + per-agent `model:`
overrides (file `.calibration`). M012 ships **schema v3** — sidecar renamed to
`.effort`, `permission_mode` field dropped, `last_preset` enum updated to
`cost | balanced | high | custom`, 6-cohort taxonomy.

v1/v2 sidecars restore byte-identically via the legacy dispatch; v2→v3
migration is triggered by `/aih-update` (silent) OR first
`/aih-effort --status` (idempotent). Schema contract: ADR-M009-A (preserved)
+ ADR-M012-A §4.


## Purpose, location, ownership (ADR-M009-A — preserved verbatim)

Sidecar survives `/aih-update` by living outside the refreshed subdirectories
(refresh loop only touches `skills/`, `agents/`, `hooks/`, `templates/`).
Records absolute restore targets so `/aih-update` can re-apply `effort:` +
`model:` frontmatter edits.

- **Location:** `.aihaus/.effort` — flat file at `.aihaus/` root alongside
  `.install-mode`, `.install-platform`, `.install-source`, `.version`.
- **User-owned, never committed.** Written by `/aih-effort` Phase 4 after
  commit + gate pass (never on `--inspect` / `--status` / self-revert fired).
  `git add .aihaus/.effort` is forbidden; `.gitignore` covers it.
- **Derived state — safe to delete.** Missing sidecar = restore silent no-op.
  Regenerated by next `/aih-effort` invocation.
- **Not source of truth.** Git history (`chore(effort):` commits) is
  authoritative; sidecar is a re-application target.

## Schema v3 (current — written by v0.16.0+)

```
# aihaus effort state — managed by /aih-effort, consumed by /aih-update
# Schema: v3 — 6-cohort taxonomy; no permission_mode field (moved to .automode)
# This file is USER-OWNED and derived state. Safe to delete. Do not commit.

schema=3
last_preset=balanced
last_commit=deadbee

# Cohort-level — written per cohort on preset/--cohort apply.
# value=custom → non-uniform; per-agent lines take over.
cohort.planner-binding.model=opus
cohort.planner-binding.effort=xhigh
cohort.planner.model=opus
cohort.planner.effort=high
cohort.doer.model=sonnet
cohort.doer.effort=high
cohort.verifier.model=haiku
cohort.verifier.effort=high
# adversarial-scout + adversarial-review intentionally absent — preset-immune.

# Per-agent overrides — applied AFTER cohort-level (overrides win).
# effort-only grammar:
implementer=high
# model grammar (dotted key):
implementer.model=sonnet
```

### v3 fields

- **`schema=3`** — mandatory first non-comment line; dispatches to v3 restore.
- **`last_preset`** — `cost | balanced | high | custom`. v2 preset names are
  renamed on migration; see `pkg/scripts/lib/restore-effort.sh` for the full
  rename map and drift guardrail logic.
- **`last_commit`** — short SHA of the `chore(effort):` commit.
- **`cohort.<name>.model`** — accepted names (6): `planner-binding`, `planner`,
  `doer`, `verifier`, `adversarial-scout`, `adversarial-review`. No colon on
  disk (`:` is CLI ergonomics only). `custom` signals non-uniform.
- **`cohort.<name>.effort`** — `low | medium | high | xhigh | max | custom`.
- **Per-agent effort** `<agent-basename>=<effort-level>` — retained from v1.
- **Per-agent model** `<agent-basename>.model=<m>` — retained from v2.

**Accepted cohort names (6):** `planner-binding`, `planner`, `doer`, `verifier`,
`adversarial-scout`, `adversarial-review`. Unknown cohort values warn + skip
(forward-compat).

## Schema-gate behavior (hard boundary — ADR-M009-A preserved)

Restore reads `schema=` field first:

- `schema=1` → legacy v1 handler (effort-only per-agent).
- `schema=2` → v2 handler (cohort-level apply first, per-agent overrides
  second — overrides win). Triggers v2→v3 migration prompt.
- `schema=3` → v3 handler (6-cohort apply + per-agent overrides).
- any other value → `warn: unknown .effort schema='X' — skipping restore` + return 0.

## Migration v2 → v3

Triggered by `/aih-update` (silent) OR first `/aih-effort --status`
(idempotent). **Authoritative implementation:** `pkg/scripts/lib/restore-effort.sh`.
The full field-by-field rename map, lossy-case warnings, and preset-rename
effort-drift guardrail (F-009, ADR-M009-A) live there — not in this annex.

Summary of migration actions:

- `.aihaus/.calibration` renamed to `.aihaus/.calibration.v2.bak` (original
  preserved; user may delete). New file: `.aihaus/.effort` (schema=3).
- `permission_mode` field dropped from output (moved to `.automode` sidecar).
- `last_preset` values renamed: v2 preset labels → v3 preset labels (see
  `restore-effort.sh` for the rename map).
- `cohort.planner.*` entries split into `planner-binding` + `planner` (with
  `!!` warning naming the split if ambiguous).
- `cohort.investigator.*` entries converted to per-agent overrides on the 3
  former investigator members (with `!!` warning).
- `cohort.adversarial.*` entries duplicated to both `adversarial-scout` and
  `adversarial-review`.
- If the v2 sidecar recorded `last_preset=<the permission-mode preset>`, the
  migration writes `.automode` with `enabled=true` + `last_enabled_at=<iso>`
  noting that the `.automode` sidecar is now orphaned (ADR-M014-A — use DSP launch via `bash .aihaus/auto.sh`)
  side effects.

## Absolute-restore semantic (ADR-M009-A — preserved verbatim)

Recorded values are verbatim restore targets, not deltas. Parse invariants:
values MUST NOT contain `=`; CRLF-stripped before use; blank + `#`-prefixed
lines skipped; whitespace-only values skipped defensively; unknown keys silently
ignored for forward-compat. To track new package defaults after an upgrade,
re-run `/aih-effort --preset <name>` — rewrites the sidecar.

## CRLF handling (Windows-compat — inherited from v1)

Bash: `value="${value%$'\r'}"` strips trailing `\r`. PowerShell:
`-replace "\r", ""` applied to every value; `Set-Content -Encoding UTF8
-NoNewline` on output frontmatter to match `sed -i.bak` byte-identically.
