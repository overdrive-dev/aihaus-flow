# SKILL Enforcement Audit

This document is the canonical audit of enforcement-layer classification for every
step in every `pkg/.aihaus/skills/aih-*/SKILL.md` (14 SKILLs) and every binding annex
(~20-25 files), generated by `tools/audit-skill-reconcile.sh --concat`.

**Authority:** ADR-260503-A (M021/2026-05-03) — locks framework, schema, move rule,
step-counting rubric, and refresh triggers. See `pkg/.aihaus/decisions.md`.

**Sole writer:** S08 implementer (per ADR-004 single-writer discipline). No other story
modifies this file after S08 commit. Regenerate via `bash tools/audit-skill-reconcile.sh --concat`.

## Framework

### 13-Column Row Schema (binding)

```
| SKILL | Location | Step | Label | Primary | Actor | Gate | Escape | Leverage | Reversibility | Drift Risk | Eligibility | Notes |
```

**Primary classification (binding enum):** `{A, B, C, A+C, B+C, A+B+C}`
- `A` model-driven — prose-only, model is actor + gate.
- `B` agent-delegated — explicit subagent spawn, agent is actor + gate.
- `C` hook-enforced — script (PreToolUse/PostToolUse/Stop/SessionStart) is actor + gate.

**Tag-set columns:**
- `actor ∈ {model, agent, hook, multi}`
- `gate ∈ {none, advisory, blocking, advisory+blocking}`
- `escape ∈ {none, opt-out-env, manual-override}`

**Score axes (binding enum):**
- `leverage ∈ {low, med, high}` — blast-radius when enforcement drifts.
- `reversibility ∈ {rev, irrev}` — irrev = requires forensic recovery once landed.
- `drift-detectability ∈ {easy, med, hard}` — easy = caught by smoke/hook/reviewer.

**Eligibility column (ADR-260502-A inheritance):**
`eligibility ∈ {deterministic, model-judgment, partial}` — gates promotion eligibility.

ADR-260502-A "Worked example #2 — DOES NOT FIT" (eligibility-gate authority, verbatim):

> **Worked example #2 — DOES NOT FIT (model judgment required).** A future maintainer wants to "auto-close" manifests where the user verbally said "I'm done with this" in a chat transcript. The eligibility condition is non-deterministic — it requires NLP/LLM judgment on free-form text. **Does NOT fit.** ADR-260502-A's pattern explicitly requires deterministic conjunction conditions; the moment a condition becomes "model decides," the architectural guarantee ("removes the model from the loop") is voided. Correct answer: this belongs in `aih-feature/SKILL.md` Step 13 prose (model-driven), not in an enforcement hook.

**Move rule (binding):** Promote A → B/C iff
`leverage=high AND (reversibility=irrev OR drift-detectability=hard) AND eligibility=deterministic`.
The third axis is the ADR-260502-A deterministic-only gate — A-classified high-leverage steps
requiring model judgment MUST STAY A (no enforcement available, by ADR).

**Refresh Triggers:** (a) new SKILL.md or annex merged, (b) existing step materially reworded,
(c) annex renamed or deleted (manual cue required — smoke ratchet does not detect rename).

**Coverage note:** `compute_expected` (S01 rubric) = 293 rows (289 M021 baseline + 4 aih-install steps added M022/Z6). Fragment total covers all rubric
steps. Cross-fragment score-consistency advisory output: `bash tools/audit-skill-reconcile.sh --score-consistency-check`.

## Scoring

Scoring axes rationale from ADR-260503-A:

- **Leverage** = magnitude of correctness-cost when enforcement drifts (blast-radius), independent
  of step size. Two implementers scoring against this prose must agree (borderline steps documented
  in Notes column).
- **Reversibility** = `irrev` once committed/landed requires forensic recovery (revert + audit);
  `rev` = rollback is clean and low-cost.
- **Drift-detectability** = `easy` caught by smoke/hook/reviewer in normal flow; `hard` surfaces
  only via incident (silent drift).

Move candidates for M022+ remediation: rows where `leverage=high AND (reversibility=irrev OR
drift-detectability=hard) AND eligibility=deterministic`. Run `bash tools/audit-skill-reconcile.sh --score-consistency-check` for cross-fragment divergences.
